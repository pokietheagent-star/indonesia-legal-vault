# SOURCE_LEDGER

This ledger tracks the legal sources used in the vault and how much confidence the team should place in them.

## Purpose
- record where a legal proposition came from,
- show whether the source is primary, secondary, or synthesized,
- track freshness and update date,
- and make every legal conclusion auditable.

## Suggested fields
- Source ID
- Source type: statute / regulation / case / doctrine / memo / internal note
- Jurisdiction / court / institution
- Title / citation
- Status: active / amended / revoked / provisional / needs review
- Freshness date
- Verified by:
- Confidence: high / medium / low
- Relevance to issue:
- Notes:

## Confidence guide
- High: sourced from primary text and verified against the current record
- Medium: sourced, but still requires careful contextual reading or cross-checking
- Low: useful for orientation only; not enough for final legal analysis

## Maintenance rules
1. Primary law should be recorded as soon as it is sourced.
2. Any change in status must be reflected here.
3. Internal memos and draft reasoning should never be mistaken for primary law.
4. Hakim should consult this ledger before final synthesis.
